# Topic Decomposition Template

Break any complex topic into learnable units before building a path.

## Decomposition Process

### Step 1 — Define "Done"

Before decomposing, state the learning goal as a capability:

```
"I want to be able to ___________."
```

Not: "I want to learn Python."  
Better: "I want to be able to write scripts that automate my file management tasks."  
Even better: "I want to be able to write a Python script that reads a CSV, filters rows by date, and outputs a summary report."

The more specific, the better the decomposition.

### Step 2 — Identify the Core Concepts

Ask: what are the essential concepts someone must understand to achieve this goal?

List them without ordering — just brainstorm. Aim for 8-15 concepts for most topics.

### Step 3 — Map Prerequisites

For each concept, ask: what do I need to know BEFORE I can understand this?

This builds the dependency graph and determines sequencing.

### Step 4 — Size Each Unit

Each learning unit should take 30-90 minutes to work through:
- Too short (< 15 min): not enough depth
- Too long (> 2 hrs): too much to retain in one session
